Validador Biométrico

Demo del modo declarativo: un integrador agrega 1 script + 1 botón a su página y obtiene validación biométrica completa con QR, polling y webhook firmado HMAC. Sin escribir JavaScript.

Esto es todo el código del frontend

<script src="https://validador-atm.duckdns.org/embed/embed-v1.js" defer></script>

<button data-validador-biometrico
        data-session-url="/api/crear-sesion"
        data-fetch-body='{"dni":"28543671"}'  <!-- DNI esperado del trámite -->
        data-on-approved="mostrarResultado">
  Validar identidad
</button>

<script>
  window.mostrarResultado = async (detail) => {
    // detail = { session_id, completed_at }
    // Fetch al backend del integrador para obtener score + PII:
    const r = await fetch(`/api/sesion-detalle?session_id=${detail.session_id}`)
    const data = await r.json()
    // data.score      = 0-100 (entero)  ← scoring agregado
    // data.confidence = "high"|"medium"|"review"|"low"  ← label
    // data.dni_extracted = { nombre, apellido, dni_number, ... }
    // data.scores = { face_similarity, liveness_score }  ← técnicos (auditoría)
  }
</script>

Probalo

⚠ En producción real este valor NO viene de un input editable — el backend del integrador lo lee de la sesión del usuario logueado. Acá lo exponemos para demostrar el cross-check: si escaneás un DNI distinto, el validador rechaza con identity_mismatch.
